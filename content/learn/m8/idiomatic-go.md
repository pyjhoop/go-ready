문법은 다 봤다. 남은 건 **관용(idiom)** — Go 개발자가 자연스럽게 여기는 방식과, 다른 언어에서 온 사람이 실제로 헛디디는 지점을 몸에 새기는 일이다. 이 레슨은 지금까지의 함정을 한자리에 모으고, 마지막 함정 하나를 더한 뒤 다음 길을 가리킨다.
{.lead}

## 마지막 함정 — goroutine 누수

`go f()`는 너무 쉬워서, **멈추는 것을 잊기 쉽다.** 채널을 영원히 기다리거나 취소 신호를 확인하지 않는 goroutine은 끝나지 못하고 메모리에 쌓인다. 크래시도 에러도 없이 서버가 서서히 무거워지는 종류의 버그다.

```go
func leak() {
    ch := make(chan int) // 아무도 보내지 않는 채널
    go func() {
        val := <-ch // 영원히 여기서 블록 — 이 goroutine은 끝나지 않는다
        fmt.Println(val)
    }()
    // 함수는 반환하지만 위 goroutine은 살아남아 샌다
}
```

방어법은 모듈 6에서 이미 배웠다: **`context`로 취소를 전파**하고, 블로킹 지점마다 `select`에 `ctx.Done()`을 함께 두어, 상위가 끝나면 하위 goroutine도 협조적으로 빠지게 한다. "이 goroutine은 언제, 무엇에 의해 끝나는가?"에 답할 수 없다면 그건 잠재적 누수다.

## 함정 총정리 — 이 코스가 지킨 약속

이 사이트의 차별점은 문법 나열이 아니라 **"어디서 미끄러지는가"**였다. 지금까지 만난 함정을 한눈에 회고한다:

{{< callout warn >}}
**경험자가 실제로 헛디디는 13가지 — 회고 체크리스트**

1. 미사용 변수·import는 경고가 아니라 **컴파일 에러**다 (1.3)
2. 숫자 타입 간 **암묵적 변환이 없다** — 전부 명시 (2.2)
3. ★ `for range` 변수 캡처 — Go 1.22 전후로 동작이 다르다 (2.4)
4. ★ 슬라이스는 백킹 배열을 **공유** → 예기치 못한 수정·append 덮어쓰기 (3.2)
5. **nil map에 쓰면 panic** (읽기는 됨) (3.3)
6. `len(s)`는 문자 수가 아니라 **바이트 수** (3.4)
7. ★ **값 리시버는 원본을 바꾸지 못한다** (4.2)
8. ★ nil 인터페이스 vs (타입 있고) 값이 nil인 인터페이스 (4.4)
9. ★ **defer의 인자는 defer 시점에 평가**되어 박제된다 (5.3)
10. main이 끝나면 goroutine도 **완료 보장 없이 죽는다** (6.1)
11. ★ 데이터 레이스 — 반드시 **`-race`로 탐지** (6.4)
12. **unexported 필드는 JSON에 나오지 않는다** (7.4)
13. **goroutine 누수** — 끝나지 못하는 goroutine이 쌓인다 (이 레슨)
{{< /callout >}}

이 13개를 반사적으로 떠올릴 수 있다면, 당신은 이미 "Go로 생각하는" 단계에 들어선 것이다. 각 함정의 공통점을 보라 — 대부분은 Go가 **비용을 숨기지 않기로** 한 결정의 결과다. 값/참조, 바이트/문자, 복사/공유의 경계를 명시적으로 드러내기 때문에, 그 경계에서 헛디디는 것이다.

## 관용의 방향

- **에러는 값이다.** `if err != nil`을 장황하다 여기지 말고, 실패 경로가 다 보이는 걸 자산으로 삼아라.
- **작은 인터페이스.** "Accept interfaces, return structs." 메서드 하나짜리 인터페이스가 유연성을 만든다.
- **통신으로 공유하라.** 단, channel이 만능은 아니다 — 단순 상태엔 Mutex가 더 맞다.
- **덜어내는 게 미덕.** 영리한 추상화보다 6개월 뒤 남이 읽을 코드. 제네릭·`init`·임베딩은 아껴 써라.
- **`gofmt`에 맡겨라.** 스타일 논쟁은 문화적으로 끝난 주제다.

## 여기서 어디로

- **Effective Go**와 **Go Code Review Comments** — 관용의 공식 문헌. 짧고 밀도 높다.
- **표준 라이브러리 소스를 읽어라.** `net/http`, `io`, `errors`의 코드 자체가 최고의 스타일 교본이다.
- **작은 실전 프로젝트** — CLI 도구, JSON API 서버, 동시성 크롤러. 배운 함정을 실제로 밟아봐야 체화된다.
- 남은 모듈 5~8의 주제(에러·동시성·표준 라이브러리·테스트)를 **한 프로젝트에 합쳐** 보는 것이 가장 빠른 정착법이다.
